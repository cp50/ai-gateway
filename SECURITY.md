# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please send an email to the repository maintainer:

- **Email:** [cp50](https://github.com/cp50) via GitHub (use the contact link on the maintainer's profile, or open a private security advisory through GitHub)

Alternatively, you can use GitHub's built-in **private vulnerability reporting** feature:

1. Go to the [Security tab](https://github.com/cp50/ai-gateway/security) of this repository.
2. Click **"Report a vulnerability"**.
3. Fill in the details and submit.

## What to Include

When reporting a vulnerability, please include:

- A description of the vulnerability and its potential impact.
- Steps to reproduce the issue.
- Any relevant logs, screenshots, or proof-of-concept code.

## Response Expectations

- **Acknowledgment:** We aim to acknowledge receipt of your report within **48 hours**.
- **Resolution:** We will work to assess and address confirmed vulnerabilities as quickly as possible, and will keep you informed of progress.

## Scope

This policy covers the `ai-gateway` codebase and its core components, including:

- API key handling and authentication logic
- Provider routing and request proxying
- Rate limiting and quota management

## Responsible Disclosure

We kindly ask that you:

- Give us reasonable time to investigate and address the issue before any public disclosure.
- Avoid exploiting the vulnerability beyond what is necessary to demonstrate it.

Thank you for helping keep this project and its users safe.
